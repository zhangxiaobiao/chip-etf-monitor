# 股票监控系统技能包

## 概述

此技能包用于快速创建专业的股票和ETF实时监控系统。系统提供实时行情获取、技术指标分析、买卖信号生成、可视化界面和价格警报功能。

## 使用场景

当以下任一情况发生时，应使用此技能：

1. 用户需要监控特定股票或ETF的实时行情
2. 需要基于技术指标生成买卖信号
3. 需要创建可视化的股票监控界面
4. 需要设置价格变动警报和通知系统
5. 需要分析历史数据并生成投资报告

## 核心功能

### 1. 数据获取模块
以至招商芯片ETF (588200.SS) 为例，系统支持：
- 实时行情数据获取
- 历史数据下载和缓存
- 多时间粒度数据支持
- 数据源降级处理

### 2. 技术分析引擎
包含以下技术指标：
- MACD（移动平均收敛发散）
- RSI（相对强弱指数）
- 布林带（Bollinger Bands）
- 移动平均线（SMA 10/20/50）
- 成交量分析

### 3. 信号生成系统
- 多指标综合判断
- 信号强度评分（0-100）
- 风险等级评估
- 具体操作建议

### 4. 可视化界面
- Streamlit Web应用
- Plotly交互式图表
- 实时仪表盘
- 技术指标图表

### 5. 通知系统
- Telegram机器人通知
- 价格变动警报
- 买卖信号警报
- 系统状态通知

## 使用方法

### 快速启动

1. **创建项目结构**：
   ```bash
   # 复制项目模板
   cp -r assets/project-template/ my-stock-monitor
   cd my-stock-monitor
   ```

2. **配置环境**：
   ```bash
   # 安装依赖
   pip install -r requirements.txt
   
   # 配置环境变量
   cp .env.example .env
   # 编辑.env文件设置股票代码和通知配置
   ```

3. **启动系统**：
   ```bash
   # 启动Web界面
   streamlit run app.py
   
   # 或启动后台监控
   python monitor.py
   ```

### 自定义配置

修改 `config.py` 文件中的以下参数：
- `STOCK_SYMBOL`: 监控的股票代码
- `MONITORING_INTERVAL`: 监控间隔（秒）
- `ALERT_THRESHOLD`: 价格警报阈值（%）
- 技术指标参数：RSI、MACD、布林带等

### 添加新的股票

1. 修改 `config.py` 中的 `STOCK_SYMBOL`
2. 更新 `STOCK_NAME` 为对应股票名称
3. 重新启动系统

## 技术栈

- **后端**: Python 3.7+
- **数据处理**: Pandas, NumPy, yfinance
- **技术指标**: TA-Lib (ta库)
- **可视化**: Plotly, Streamlit
- **通知**: python-telegram-bot
- **任务调度**: schedule

## 扩展功能

### 添加新的技术指标
在 `signals.py` 中添加新的指标计算方法，并在 `generate_signals` 函数中集成。

### 支持多个股票
修改数据获取模块，支持同时监控多个股票，并在界面中显示切换选项。

### 数据库集成
添加数据库支持，存储历史数据和信号记录，便于回测和分析。

### 移动端适配
使用响应式设计或创建移动端专用界面。

## 故障排除

### 数据获取失败
1. 检查网络连接
2. 验证股票代码格式
3. 查看数据源API状态

### 依赖包安装失败
1. 使用Python 3.7或更高版本
2. 尝试使用 `--user` 参数安装
3. 检查pip版本：`python -m pip install --upgrade pip`

### 界面无法启动
1. 确保Streamlit已正确安装
2. 检查端口8501是否被占用
3. 查看浏览器控制台错误信息

## 最佳实践

1. **配置管理**: 将所有可配置参数放在 `config.py` 中

5. **测试验证**: 定期测试核心功能和数据准确性

## 性能优化建议

1. **数据缓存**: 使用本地缓存减少API调用
2. **异步处理**: 对耗时操作使用异步编程
3. **资源管理**: 及时释放不再使用的资源
4. **代码优化**: 使用向量化操作替代循环

## 安全注意事项

1. **API密钥**: 不要将API密钥硬编码在代码中
2. **环境变量**: 使用环境变量存储敏感信息
3. **输入验证**: 对所有外部输入进行验证
4. **错误信息**: 避免泄露敏感信息的错误消息
